1. **Create `src/lib/clipboard-handler.ts`** with three exported pure functions:

   a. `parseClipboardText(text: string): { rows: string[][]; rowCount: number; colCount: number }`:
      - Split by `\n`, then each row by `\t`.
      - Trim whitespace from each cell.
      - Filter out completely empty rows.
      - Return row/col counts.

   b. `parseNumericValue(value: string): number | null`:
      - Empty string returns 0 (clear cell).
      - Handle European comma decimal (`120,5` -> `120.5`).
      - Validate: `isNaN`, `< 0`, `> 999` return `null`.
      - Round to integer (hours are whole numbers).

   c. `mapPasteToGridCells(parsed, focusedRowIndex, focusedColIndex, gridRows, months, currentMonth)`:
      - `parsed`: `{ rows: string[][] }` from `parseClipboardText`.
      - `focusedRowIndex`: number (0-based row in grid data).
      - `focusedColIndex`: number (0-based index into the `months` array).
      - `gridRows`: `GridRow[]` (the data rows, excluding add-row and pinned).
      - `months`: `string[]` (all month column IDs).
      - `currentMonth`: `string` (YYYY-MM).
      - For each cell in parsed data, compute target row/col. Skip if:
        - Target row >= gridRows.length (out of bounds)
        - Target row has `projectId` starting with `__` (special rows)
        - Target month < currentMonth (past, read-only -- per Pitfall 3)
      - Parse each value with `parseNumericValue`. If null, add to `errors[]` with row/col/value/reason.
      - Return `{ cells: PasteCell[], errors: PasteError[], skippedReadOnly: number }`.

   Export types: `PasteCell = { projectId: string; month: string; hours: number }`, `PasteError = { row: number; col: number; value: string; reason: string }`.

2. **Create `src/hooks/use-keyboard-nav.ts`** with two exported pure functions:

   a. `tabToNextCell(params: TabToNextCellParams): CellPosition | boolean`:
      - If `nextCellPosition` is null, return false (at grid boundary).
      - If `nextCellPosition.rowPinned` is not null, return false (skip pinned rows).
      - Otherwise return `nextCellPosition`.

   b. `navigateToNextCell(params: NavigateToNextCellParams): CellPosition | null`:
      - If `nextCellPosition` is null, return `previousCellPosition` (stay put at boundary).
      - If `nextCellPosition.rowPinned` is not null, return `previousCellPosition` (skip pinned).
      - Otherwise return `nextCellPosition`.

   Note: Enter key behavior is handled by `enterNavigatesAfterEdit={true}` on the grid, NOT in `navigateToNextCell`. Do NOT add special Enter handling here (Pitfall 6).

   Import types from `ag-grid-community`: `TabToNextCellParams`, `NavigateToNextCellParams`, `CellPosition`.
  </action>

  <action>
1. **Add keyboard navigation props** to the `AgGridReact` component in `allocation-grid.tsx`:
   - Import `tabToNextCell` and `navigateToNextCell` from `@/hooks/use-keyboard-nav`.
   - Add props: `tabToNextCell={tabToNextCell}`, `navigateToNextCell={navigateToNextCell}`.
   - Add `enterNavigatesAfterEdit={true}` -- this makes Enter commit the edit AND move down (Excel behavior). Do NOT use `enterNavigatesVertically` (research note was uncertain; `enterNavigatesAfterEdit` alone is correct for AG Grid 35.x).

2. **Add paste event listener** to the grid container:
   - Add a `gridContainerRef = useRef<HTMLDivElement>(null)` for the outer wrapper div (the one with `className="h-[600px] w-full"`).
   - Attach the ref to that div.
   - In a `useEffect`, add a `paste` event listener to `gridContainerRef.current`.
   - The paste handler:
     a. Get text from `e.clipboardData?.getData('text/plain')`. If no text, return.
     b. `e.preventDefault()`.
     c. Get `gridRef.current` (the GridApi). If null, return.
     d. Get focused cell: `gridRef.current.getFocusedCell()`. If null, return.
     e. Get `colId = focusedCell.column.getColId()`. Find `monthIndex = months.indexOf(colId)`. If -1 (focused on projectName column), return.
     f. Get the row index. The `focusedCell.rowIndex` is the display row index. Filter `localRowData` to get data rows only (exclude isAddRow).
     g. Call `parseClipboardText(text)`.
     h. Call `mapPasteToGridCells(parsed, focusedCell.rowIndex, monthIndex, dataRows, months, currentMonth)`.
     i. If `errors.length > 0`, log a warning or show a console message (toast integration deferred -- no toast library yet).
     j. If `skippedReadOnly > 0`, log: `${skippedReadOnly} cells skipped (past months are read-only)`.
     k. For each cell in `cells`, call `onCellChange({ projectId, month, hours })` AND update `localRowData` state to reflect the pasted values immediately in the grid.
     l. Update localRowData in a single `setLocalRowData` call that applies all pasted values at once (batch update, not one-by-one).
   - Cleanup: remove event listener on unmount.
   - Dependency array: `[gridRef, months, currentMonth, localRowData, onCellChange]`. Use a ref for `localRowData` to avoid re-attaching the listener on every row change (stale closure prevention).

3. **Make grid container focusable** for paste to work:
   - Add `tabIndex={0}` to the wrapper div so it can receive focus and paste events.
   - Add `outline-none` to prevent focus ring on the container (AG Grid cells have their own focus styling).
