1. **Extend `PersonFilter` type** in `person.types.ts`: add `withStatus?: boolean` field. Add a new exported type `PersonWithStatus`:
```typescript
export type PersonWithStatus = PersonRow & {
  departmentName: string;
  disciplineAbbreviation: string;
  status: CapacityStatus;
  currentMonthSum: number;
};
```

2. **Add `listPeopleWithStatus` function** to `person.service.ts`:
   - Accept `orgId` and `filters` (same as `listPeople`).
   - Query people table joined with departments (for `name`), disciplines (for `abbreviation`), and LEFT JOIN allocations filtered to current month (`getCurrentMonth()` from `@/lib/date-utils`, normalized to `YYYY-MM-01` for DB comparison).
   - GROUP BY person fields, SUM(allocations.hours) as `currentMonthSum`.
   - Compute `status` using `calculateStatus(currentMonthSum, targetHoursPerMonth)` from `@/lib/capacity`.
   - Return `PersonWithStatus[]` ordered by `sortOrder, lastName, firstName`.
   - This is a SINGLE query with JOIN and GROUP BY, NOT N+1.

3. **Add `getAdjacentPerson` function** to `person.service.ts`:
   - Accept `orgId`, `currentPersonId`, `direction: 'next' | 'prev'`, and optional `filters: PersonFilter`.
   - Call `listPeople(orgId, filters)` to get the full ordered list.
   - Find `currentPersonId` in the list, return the next or previous person's `{ id }`, or `null` if at boundary.
   - This simple approach is fine because the people list is small (tens, not thousands).

4. **Modify `GET /api/people` route** to support `?withStatus=true` query param:
   - If `withStatus=true`, call `listPeopleWithStatus` instead of `listPeople`.
   - Return `{ people: PersonWithStatus[] }`.
   - Otherwise, existing behavior unchanged.

5. **Create `GET /api/people/[id]/adjacent` route**:
   - Accept query param `direction=next|prev`.
   - Call `getAdjacentPerson(orgId, id, direction)`.
   - Return `{ person: { id } | null }`.
   - Use `getTenantId()` for org scoping, `requireRole('viewer')` for auth.
