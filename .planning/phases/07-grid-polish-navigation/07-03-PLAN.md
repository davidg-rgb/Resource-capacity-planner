Add custom drag-to-fill functionality and conflict detection to the allocation grid.

Purpose: Drag-to-fill enables rapid data entry by copying a cell value across months. Conflict detection prevents data loss when two users edit the same allocation simultaneously. Both are critical for a production-grade spreadsheet experience. AG Grid's fill handle is Enterprise-only, so this is a custom implementation.

Output: DragToFillHandle overlay component, extended autosave with conflict detection, server-side conflict checking.
</objective>

3. **Modify `batchUpsertAllocations`** in `allocation.service.ts`:
   - Accept extended `AllocationUpsert[]` with optional `expectedUpdatedAt`.
   - Before each upsert, if `expectedUpdatedAt` is provided:
     a. SELECT the existing allocation's `updatedAt` and `hours` for that `(orgId, personId, projectId, month)`.

   - Return the existing `BatchUpsertResult` with the new `conflicts` array.
   - Also return `updatedTimestamps: Record<string, string>` mapping `"projectId:month"` -> new `updatedAt` ISO string for each successful upsert.

   - Export a `setUpdatedAt(key: string, value: string)` function for initializing from fetched allocations.
   - Export a `initUpdatedAtFromAllocations(allocations: FlatAllocation[])` helper that populates the map from the allocations query result.
   - In the `flush` function, include `expectedUpdatedAt` from the map for each pending upsert.
   - On successful response:
     a. If `conflicts.length > 0`: log conflict details to console. Call a conflict callback (add `onConflict?: (conflicts: ConflictInfo[]) => void` to hook params or return value). The callback shows a simple `window.confirm("Cell X was modified by another user. Overwrite?")` -- if yes, re-send those cells without `expectedUpdatedAt` (force). If no, invalidate query to refetch.
     b. Update `updatedAtMap` with returned `updatedTimestamps` (Pitfall 4 prevention -- update map after OWN saves so subsequent saves don't false-positive).
   - On error: invalidate as before.

   - The route at `src/app/api/allocations/batch/route.ts` should pass `expectedUpdatedAt` from request body through to the service.

1. **Create `DragToFillHandle` component** at `src/components/grid/drag-to-fill-handle.tsx`:

   Props:
   ```typescript
   type DragToFillHandleProps = {
     gridApi: GridApi | null;
     months: string[];
     currentMonth: string;
     localRowData: GridRow[];
     onFill: (cells: { projectId: string; month: string; hours: number }[]) => void;
   };
   ```

   Implementation:
   - Track focused cell using AG Grid events. Listen for `cellFocused` event on `gridApi.addEventListener('cellFocused', ...)`.
   - When a cell is focused:
     a. Get the column ID. If it's `projectName` or the cell is in a pinned row, hide the handle.
     b. Check if the month is >= currentMonth (editable). If not, hide.
     c. Get the focused cell's DOM element: use `gridApi.getCellRendererInstances({ columns: [colId], rowNodes: [rowNode] })` OR find the cell via DOM query `[col-id="${colId}"][row-index="${rowIndex}"]` on the grid container.
     d. Position the handle (a 10x10px blue square, `bg-primary cursor-crosshair`) at the bottom-right corner of the cell using absolute positioning relative to the grid container.
   - On `mousedown` on the handle:
     a. Record the source cell's value, projectId, and month index.
     b. Set `isDragging = true` (use a ref, NOT state, for performance -- Pitfall: mousemove fires 60+/sec).
   - On `mousemove` (attached to `document` during drag):
     a. Determine which month column the mouse is over by computing column boundaries.
     b. Highlight the range from source month to current month by overlaying a semi-transparent `bg-primary/20` div across the cells.
     c. HORIZONTAL ONLY -- same row (same project), different months. Do not support vertical drag.
     d. Only include months >= currentMonth in the range.
   - On `mouseup` (attached to `document` during drag):
     a. Compute all cells in the fill range: same projectId, each month from source+1 to end of drag.
     b. Each cell gets the source cell's hours value.
     c. Call `onFill(cells)` with the array.
     d. Clean up: remove highlight, set isDragging false.
   - On grid scroll (`gridApi.addEventListener('bodyScroll', ...)`): hide the handle during scroll, reposition after scroll ends.
   - Cleanup: remove all event listeners on unmount.

   Render: A single absolutely-positioned div (the handle square) + a range highlight div (visible during drag). Both positioned relative to the grid container.

   - The `onFill` callback should:
     a. Update `localRowData` with all filled values in one `setLocalRowData` call.
     b. Call `onCellChange` for each filled cell to trigger autosave.
   - Make the grid container div `position: relative` so the handle can use absolute positioning.
