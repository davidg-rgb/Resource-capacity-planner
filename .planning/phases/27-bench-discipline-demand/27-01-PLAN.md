# Phase 27 Plan 01: Bench Report + Discipline Demand Heatmap Widgets

## Objective

Build two analytical table-style visualization widgets (V8 and V12) that implement WidgetProps, consume existing data hooks, and register in the widget registry.

## Context

- Spec: `.planning/DASHBOARD-VISUALIZATIONS-SPEC.md` (V8 section line 635, V12 section line 916)
- Widget pattern: `src/features/dashboard/widgets/kpi-cards-widget.tsx`
- Data hooks: `src/hooks/use-bench-report.ts`, `src/hooks/use-discipline-demand.ts`
- Types: `src/features/analytics/analytics.types.ts` (BenchReportResponse, DisciplineDemandResponse)

## Tasks

### Task 1: V8 Bench Report Widget
type="auto"

Create `src/features/dashboard/widgets/bench-report-widget.tsx`:
- Summary KPI cards (bench hours, FTE equivalent, people count)
- Trend indicator vs previous period
- Department breakdown table with inline bar charts (Tailwind width%)
- Discipline breakdown table with inline bar charts
- Top available people list (top 5)
- Insight callout section
- Loading/error/empty states
- Register with widget registry (id: 'bench-report', category: 'health-capacity', defaultColSpan: 12)

**Done:** Component renders all spec sections, handles loading/error, registered in registry.

### Task 2: V12 Discipline Demand Heatmap Widget
type="auto"

Create `src/features/dashboard/widgets/discipline-demand-widget.tsx`:
- Month x discipline grid table
- Sub-rows: demand, supply, gap per cell
- Status color cells: surplus=green, balanced=blue, tight=amber, deficit=red
- Sustained deficit hiring indicator (3+ consecutive months)
- Summary section with combined deficit and FTE hiring need
- Loading/error/empty states
- Register with widget registry (id: 'discipline-demand', category: 'timelines-planning', defaultColSpan: 12)

**Done:** Component renders heatmap grid with status colors, hiring indicators, registered in registry.

### Task 3: Register in Widget Index
type="auto"

Update `src/features/dashboard/widgets/index.ts` to import both new widgets.

**Done:** Both widgets imported and auto-registered on app load.

## Verification

- TypeScript compiles without errors
- Both widgets exported and registered
- Widget IDs match expected naming convention
