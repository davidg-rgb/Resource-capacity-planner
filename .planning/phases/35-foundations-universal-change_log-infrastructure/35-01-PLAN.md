<objective>
Phase 35 — stand up the universal `change_log` audit spine. Deliver the `change_log` table + drizzle migration, the single-writer `recordChange()` helper, and the three-mechanism enforcement (custom eslint rule + deterministic codegen manifest + runtime invariant test) so every v5.0 mutating service — starting in Phase 37 — is guaranteed to emit an audit row inside its transaction.

This is noted in the ROADMAP Phase 36 entry (update that separately if needed; this plan only claims the table).

       import { z } from 'zod';
       import { db } from '@/db';
       import { changeLog, changeLogActionEnum, changeLogEntityEnum } from './change-log.schema';
       import type { RecordChangeInput, ChangeLogEntry } from './change-log.types';
       import type { PgTransaction } from 'drizzle-orm/pg-core';

       const recordChangeInputSchema = z.object({
         orgId: z.string().uuid(),
         actorPersonaId: z.string().min(1),
         entity: z.enum(changeLogEntityEnum.enumValues),
         entityId: z.string().uuid(),
         action: z.enum(changeLogActionEnum.enumValues),
         previousValue: z.unknown().nullable(),
         newValue: z.unknown().nullable(),
         context: z.record(z.unknown()).nullable(),
       });

       export async function recordChange(
         input: RecordChangeInput,
         tx?: Pick<typeof db, 'insert'>,
       ): Promise<ChangeLogEntry> {
         const parsed = recordChangeInputSchema.parse(input);
         const executor = tx ?? db;
         const [row] = await executor
           .insert(changeLog)
           .values({

             actorPersonaId: parsed.actorPersonaId,
             entity: parsed.entity,

             action: parsed.action,
