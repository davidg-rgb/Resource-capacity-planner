# Phase 30 Plan 01: Integration & Wiring

## Objective

Wire all 13 new visualization widgets + existing widgets together: verify registry completeness, fix default layout widget ID mismatches, create project leader dashboard route, add cross-linking actions, enhance widget drawer with search, and provide mobile layout defaults.

## Context

- All 18 widgets (7 existing + 11 new) are registered via side-effect imports in widgets/index.ts
- Default layouts reference IDs `stacked-area-chart`, `allocation-trends`, `discipline-distribution` that don't match any registered widget
- Capacity forecast chart already supports `onDeficitClick` callback
- Person 360 overlay is globally available via `usePersonCard()`

## Tasks

### Task 1: Fix default layout widget ID mismatches and add new widgets
- Fix `stacked-area-chart` -> `capacity-distribution` in default-layouts.ts
- Create `allocation-trends-widget.tsx` and `discipline-distribution-widget.tsx` wrapper widgets
- Update manager desktop default to include new v4 widgets (gauges, sparklines, forecast, bench, discipline-demand, conflicts, period-comparison)
- Update project-leader desktop default with correct IDs and new widgets (program-rollup, resource-conflicts)

### Task 2: Create mobile layout defaults
- Manager mobile: all widgets full-width (colSpan 12), essential subset
- Project-leader mobile: all widgets full-width, essential subset

### Task 3: Wire cross-linking actions
- Create a cross-linking context/event bus for widget-to-widget communication
- Capacity Forecast: on deficit click -> scroll to / highlight availability finder with month pre-filled
- Resource Conflicts: click person name -> open Person 360 overlay
- Availability Timeline: click person -> open Person 360 overlay

### Task 4: Create project leader dashboard route
- New route at `/dashboard/projects/page.tsx`
- Project selector dropdown at top
- Uses `DashboardGrid` with `dashboardId="project-leader"`
- Add to side-nav and top-nav

### Task 5: Widget drawer search enhancement
- Add search input at top of WidgetDrawer
- Filter widgets by name/description as user types
- Maintain category grouping in results

## Verification

- All 18 widget IDs in default layouts match registered widget IDs
- `/dashboard/projects` renders with project-leader default layout
- Widget drawer shows search bar, categories, and filters correctly
- Cross-linking: forecast deficit click scrolls to finder section
- Mobile layouts exist for both personas

## Success Criteria

- Zero console warnings about missing widget IDs
- Both dashboard personas fully functional with all relevant widgets
- Cross-linking provides seamless navigation between related widgets
