# Phase 32 Plan 01: What-If Scenario Mode

## Objective
Build isolated What-If Scenario mode enabling managers to explore hypothetical staffing plans without affecting production data. Complete isolation from existing code paths.

## Context
- Spec: V13 section of DASHBOARD-VISUALIZATIONS-SPEC.md
- UX Review: V13-SCENARIO-UX-REVIEW.md (all decisions resolved)
- Architecture: Dedicated route, separate DB tables, separate cache namespace

## Tasks

### Task 1: DB Schema (type="auto")
Add 3 new tables to schema.ts: scenarios, scenario_allocations, scenario_temp_entities with proper indexes, enums, and relations.

### Task 2: Feature Flag (type="auto")
Add 'scenarios' to FLAG_NAMES, FeatureFlags, FLAG_ROUTE_MAP, default flags, and platform admin labels.

### Task 3: Scenario Service Layer (type="auto")
Create scenario.service.ts with CRUD, allocation management, promote-to-actual, and temp entity operations. Create scenario-analytics.service.ts mirroring analytics.service.ts but reading from scenario_allocations.

### Task 4: API Routes (type="auto")
Create 9 API routes under /api/scenarios/: list, create, get, update, delete, allocations CRUD, analytics (dashboard, team-heatmap, impact, comparison), promote, temp-entities.

### Task 5: Client Hooks (type="auto")
Create 7 hooks with ['scenario', scenarioId, ...] cache namespace: use-scenarios, use-scenario-allocations, use-scenario-analytics, use-scenario-promote, use-scenario-grid-autosave, use-scenario-temp-entities.

### Task 6: UI Components (type="auto")
Create scenario-banner (non-dismissible amber), scenario-card, create-scenario-modal, impact-bar, promote-modal (two-step with confirmation friction), exit-scenario-dialog, comparison-view.

### Task 7: Pages (type="auto")
Create /scenarios list page, /scenarios/[id] editor page, /scenarios/[id]/compare side-by-side page.

### Task 8: Navigation (type="auto")
Add scenarios nav item to top-nav (flag-gated, FlaskConical icon, between Overview and Warnings) and side-nav section.

### Task 9: i18n (type="auto")
Add Swedish and English translations for nav and sidebar scenario entries.

## Verification
- TypeScript compiles with zero errors
- ESLint passes with zero errors on all new files
- Zero modifications to existing service functions, hooks, or components (only schema, flags, nav, platform admin)
- All scenario hooks use ['scenario', ...] query key prefix
- All scenario API routes under /api/scenarios/
- Banner is non-dismissible on all /scenarios/* pages
- Promote is admin-gated (requireRole('admin'))

## Success Criteria
- Feature flag gates access to /scenarios route and nav item
- Creating a scenario snapshots current allocations
- Editing scenario allocations writes ONLY to scenario_allocations
- Comparison view shows actual vs scenario side-by-side
- Promote flow is row-level selective with confirmation friction
- Zero cache contamination between scenario and actual data
