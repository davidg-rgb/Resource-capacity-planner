**2. Add three functions to `src/features/analytics/analytics.service.ts`** (append, do NOT modify existing getTeamHeatMap):

`getDashboardKPIs(orgId: string, monthFrom: string, monthTo: string): Promise<DashboardKPIs>`
- Single CTE query with generate_series (same pattern as getTeamHeatMap)
- person_utilization CTE: JOIN people x month_series, SUM allocations grouped by person
- Final SELECT: COUNT people, SUM allocated / SUM target for utilization %, COUNT FILTER for over (>100%) and under (<50%)
- Guard division by zero: if total_target is 0, return utilizationPercent: 0
- Reference the exact SQL pattern from RESEARCH.md Pattern 1

`getDepartmentUtilization(orgId: string, monthFrom: string, monthTo: string): Promise<DepartmentUtilization[]>`
- CTE with generate_series, group by department
- For each department: SUM(allocated_hours) / SUM(target_hours * month_count) * 100
- Guard division by zero per department
- ORDER BY departmentName ASC
- Return array of { departmentId, departmentName, utilizationPercent }

`getDisciplineBreakdown(orgId: string, monthFrom: string, monthTo: string): Promise<DisciplineBreakdown[]>`
- CTE with generate_series, JOIN people -> disciplines, SUM allocation hours by discipline
- Return array of { disciplineId, disciplineName, totalHours }
- ORDER BY totalHours DESC (most-used discipline first)
- People with no discipline are excluded (INNER JOIN disciplines)

All functions: filter by `p.organization_id = orgId::uuid` and `p.archived_at IS NULL` (active people only, same as getTeamHeatMap).
