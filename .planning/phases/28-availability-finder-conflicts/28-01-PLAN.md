# Phase 28 Plan 01: Availability Finder + Resource Conflicts (C3)

## Objective

Build two interactive visualization widgets with action capabilities:
- V3 Availability Finder Panel: search/filter with ranked results and quick-assign
- V9 Resource Conflict Matrix: overloaded people with resolution tools

## Context

- Spec: `.planning/DASHBOARD-VISUALIZATIONS-SPEC.md` (V3 and V9 sections)
- Widget pattern: `src/features/dashboard/widgets/kpi-cards-widget.tsx`
- Data hooks: `src/hooks/use-availability.ts`, `src/hooks/use-conflicts.ts`
- Types: `src/features/analytics/analytics.types.ts` (AvailabilitySearchResponse, ConflictsResponse)
- Batch API: `POST /api/allocations/batch` with AllocationUpsert payload

## Tasks

### Task 1: Quick-Assign Modal (shared component)
- type: auto
- Create `quick-assign-modal.tsx` in widgets directory
- Project selector dropdown (useProjects hook)
- Month range inputs with per-month hours breakdown
- Uniform hours "Apply to all" shortcut
- Batch allocation via POST /api/allocations/batch
- Invalidates availability-search, conflicts, allocations, alerts queries

### Task 2: V3 Availability Finder Widget
- type: auto
- Create `availability-finder-widget.tsx`
- Filter bar: discipline, department, min hours, sort order
- Results list with person name (triggers Person 360 Card), discipline, department
- Per-month mini utilization bars (allocated vs target)
- Total available hours summary per person
- "Assign" button per person opens QuickAssignModal
- Empty state with helpful message
- Register as 'availability-finder' in widget registry (alerts-actions category)

### Task 3: V9 Resource Conflict Widget
- type: auto
- Create `resource-conflict-widget.tsx`
- Current month vs next 3 months toggle
- Conflict cards sorted by severity (most overallocated first)
- Per-project horizontal bars showing allocation vs target
- Suggested resolution display with one-click apply
- "Redistribute Manually" modal with per-project hour inputs, total validation
- "Dismiss" to localStorage (acknowledged conflicts section)
- History footer: resolved vs pending counts
- Person names trigger Person 360 Card
- Register as 'resource-conflicts' in widget registry (alerts-actions category)

### Task 4: Widget Registry Integration
- type: auto
- Add imports in `widgets/index.ts`
- Verify TypeScript compilation

## Verification

- `npx tsc --noEmit` passes
- Both widgets registered and visible via getAllWidgets()
- Quick-assign modal handles project selection, month range, save flow
- Conflict widget shows dismiss, apply suggestion, redistribute flows

## Success Criteria

- 3 new component files (quick-assign-modal, availability-finder-widget, resource-conflict-widget)
- Both widgets implement WidgetProps interface
- Both registered in widget-registry
- Person names wire to Person 360 Card via usePersonCard
- All actions use existing batch allocation API
