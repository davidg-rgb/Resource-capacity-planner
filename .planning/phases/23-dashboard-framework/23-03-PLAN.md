**Step 1:** Create `src/app/api/analytics/person-summary/route.ts` (R23-08):

Follow the exact pattern from `src/app/api/analytics/alerts/route.ts`:

**GET handler:**
- Extract `orgId` via `getTenantId()`
- Read query param `personId` (required, return 400 if missing)
- Query the database for the person's summary data (tenant-scoped: verify person belongs to orgId):
  - Person basic info: firstName, lastName, email, department name, discipline names
  - Current utilization percentage (from allocations in current month range)
  - Active allocations: list of `{ projectName, role, percentage, startDate, endDate }`

  ```typescript
  {
    id: string;
    firstName: string;
    lastName: string;
    email: string | null;
    department: { id: string; name: string } | null;
    disciplines: { id: string; name: string }[];
    utilizationPercent: number;
    capacityStatus: 'available' | 'fully-allocated' | 'overloaded';
    activeAllocations: {
      projectId: string;
      projectName: string;
      role: string | null;
      percentage: number;
      startDate: string;
      endDate: string;
    }[];
    totalFteEquivalent: number;
  }
  ```

**Step 1:** Create `src/features/dashboard/person-card/person-card-provider.tsx` (R23-07):

`'use client'` directive.

**PersonCardContext:**
- State: `{ personId: string | null, isOpen: boolean }`
- Methods: `openPersonCard(personId: string): void`, `closePersonCard(): void`
- Context value: `{ personId, isOpen, openPersonCard, closePersonCard }`

**PersonCardProvider component:**
- Maintains `personId` state (null = closed)
- `openPersonCard` sets personId and isOpen=true
- `closePersonCard` sets personId=null and isOpen=false
- Renders children + conditionally renders `PersonCardOverlay` via `React.createPortal` (portal target: `document.body`)
- Only render portal on client side (check `typeof document !== 'undefined'`)

**Export `usePersonCard()` hook** that reads context, throws if used outside provider.

**Step 2:** Create `src/features/dashboard/person-card/person-card-overlay.tsx` (R23-07):

`'use client'` directive.

**PersonCardOverlay component** — receives `personId` and `onClose` props:
- Fixed position panel: `fixed top-0 right-0 h-full w-[360px]` with z-60 (per research z-index hierarchy: above widget drawer z-50, below modals z-70)
- Backdrop: semi-transparent overlay `fixed inset-0 bg-black/20 z-[59]`
- Slide-in animation: CSS transform `translateX(100%)` -> `translateX(0)` with `transition-transform duration-300`
- Use `usePersonSummary(personId)` to fetch data
- **Close triggers:**
  - Click on backdrop calls `onClose`
  - Escape key: `useEffect` with `keydown` listener for Escape
  - X button in top-right corner of panel
- **Content layout** (scrollable body):
  - Header: Full name (large), capacity status badge (colored: green=available, yellow=fully-allocated, red=overloaded)
  - Info section: Department, disciplines (tags), email, utilization % with small bar
  - Allocations section: List of active allocations with project name, role, percentage bar, date range
  - Footer: "View full profile" link to `/people/{id}` (or wherever person detail lives)
- Loading state: skeleton with 3 pulsing blocks
- Error state: "Failed to load person data" with retry button
- Use Tailwind classes consistent with project design tokens (bg-surface, text-on-surface, etc.)
