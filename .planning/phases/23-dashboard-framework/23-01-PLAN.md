**R23-04:** Add the `dashboardLayouts` table to `src/db/schema.ts`.

Add after the existing table definitions (before relations section):

```typescript
export const dashboardLayouts = pgTable(
  'dashboard_layouts',
  {
    id: uuid('id').primaryKey().defaultRandom(),
    organizationId: uuid('organization_id')
      .notNull()
      .references(() => organizations.id),
    clerkUserId: text('clerk_user_id'),  // null = tenant default
    dashboardId: text('dashboard_id').notNull(), // 'manager' | 'project-leader'
    deviceClass: text('device_class').notNull().default('desktop'),
    layout: jsonb('layout').notNull(),  // WidgetPlacement[]
    version: integer('version').notNull().default(1),
    createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
    updatedAt: timestamp('updated_at', { withTimezone: true })
      .notNull()
      .defaultNow()
      .$onUpdate(() => new Date()),
  },
  (table) => [
    uniqueIndex('uq_dashboard_layout_user').on(
      table.organizationId,
      table.clerkUserId,
      table.dashboardId,
      table.deviceClass,
    ),
  ],
);
```

**IMPORTANT — NULL unique constraint pitfall:** PostgreSQL does NOT enforce uniqueness on NULL values in standard unique indexes. To prevent duplicate tenant defaults (where clerk_user_id IS NULL), add a partial unique index for tenant defaults. Use Drizzle's `index()` with a raw SQL expression if needed, or use the sentinel value approach: store `'__tenant_default__'` instead of NULL for tenant defaults. Choose the sentinel approach as it is simpler with Drizzle — document the convention with a comment in the schema.

After adding the table, run:
```bash
pnpm drizzle-kit generate
pnpm drizzle-kit push
```

If `drizzle-kit push` is not the project convention, check for `db:push` or `db:generate` scripts in package.json and use those instead.
