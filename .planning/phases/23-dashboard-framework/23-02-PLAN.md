**Step 1:** Create `src/app/api/dashboard/layout/route.ts` (R23-05, R23-06):

**GET handler:**
- Extract orgId via `getTenantId()`, userId via `auth()` (from @clerk/nextjs/server)
- Read query params: `dashboardId` (default 'manager'), `deviceClass` (default 'desktop')
- Implement 4-tier layout resolution (R23-06):
  1. Query `dashboardLayouts` for personal layout (matching orgId + userId + dashboardId + deviceClass)
  2. If not found, query for same user but opposite deviceClass (device clone tier)
  3. If not found, query for tenant default (clerkUserId = '__tenant_default__' + same dashboardId + deviceClass)
  4. If not found, call `getDefaultLayout(dashboardId, deviceClass)` from default-layouts.ts
- Return `{ source: 'personal' | 'cloned' | 'tenant-default' | 'built-in', layout: DashboardLayoutData }`
- Filter out widget IDs not in current registry (pitfall 3 from research): call `getWidget(widgetId)` for each placement, skip if undefined

**PUT handler:**
- Extract orgId + userId, validate body with Zod schema: `{ dashboardId: string, deviceClass: string, widgets: WidgetPlacement[] }`
- Upsert into dashboardLayouts using Drizzle `onConflictDoUpdate` on the unique index
- Return 200 with saved layout

**Step 2:** Create `src/app/api/dashboard/layout/default/route.ts` (R23-05):

**PUT handler:**
- Use `requireRole('admin')` for authorization
- Validate body: `{ dashboardId: string, deviceClass: string, widgets: WidgetPlacement[] }`
- Upsert with clerkUserId = '__tenant_default__'
- Return 200

- Use `optimisticUpdate` pattern: update query cache immediately, roll back on error (show toast via sonner `import { toast }`)

**Step 1:** Create `src/features/dashboard/dashboard-layout-engine.tsx` (R23-02):

`'use client'` directive. This is the main grid renderer.

**DashboardGrid component** — props: `{ dashboardId?: string }`:
- Call `useDashboardLayout(dashboardId)` to get current layout
- Call `useWidgetTimeRange` for global time range
- Maintain local `isEditMode` state (boolean, default false)
- Maintain local `widgets` state (WidgetPlacement[]) synced from layout query data
- Wrap grid in `DndContext` from @dnd-kit/core with `closestCenter` collision detection
- Inside: `SortableContext` with `items={widgets.map(w => w.widgetId)}` and `strategy={() => null}` (CRITICAL: bypass default strategy for variable-span grid, per research pitfall 1)
- Render `<div className="grid grid-cols-12 gap-6">` containing mapped widgets
- Each widget renders via `<SortableWidget>` (defined below)
- Include `<DragOverlay>` that renders the currently dragged widget at fixed width matching its rendered width

**SortableWidget component** — uses `useSortable` from @dnd-kit/sortable:
- Wraps the widget component from registry (`getWidget(placement.widgetId)?.component`)
- Applies `style={{ gridColumn: \`span ${placement.colSpan}\` }}`
- In edit mode: shows drag handle (GripVertical icon), resize handle, remove button via overlay div on top (per research pitfall 6 — do NOT pass isEditMode into widget render logic for data layer, only for chrome)
- `React.memo` the inner widget component to prevent re-renders when entering/exiting edit mode

**onDragEnd handler:**
- Reorder widgets array based on active/over IDs
- Call `useSaveLayout` mutation with new order
- Use `arrayMove` from @dnd-kit/sortable for reorder logic

**Step 2:** Create `src/features/dashboard/dashboard-edit-mode.tsx` (R23-03):

**EditModeToggle component:**
- Pencil icon button (Pencil from lucide-react) in top-right of dashboard
- Toggles `isEditMode` state (passed as prop from DashboardGrid)
- When active: shows blue highlight ring, "Done" text instead of pencil

**WidgetDrawer component:**
- Slides in from right (CSS transform `translateX`, transition 300ms)
- Lists all available widgets for current dashboard via `getWidgetsByDashboard(dashboardId)`
- Each widget shows: icon, name, description, defaultColSpan badge
- Click adds widget to layout (appends to widgets array at next position)
- Widgets already in layout show "Added" badge and are not clickable
- Group by `category` with section headers

**ResizeHandle component:**
- Small handle at bottom-right corner of widget in edit mode
- Click cycles through valid colSpan values: 4 -> 6 -> 12 -> 4 (but never below minColSpan from WidgetDefinition)
- Shows current span as small badge "4/12", "6/12", "12/12"

**RemoveButton component:**
- X icon at top-right of widget in edit mode
- On click: removes widget from layout array, saves via mutation

**Z-index hierarchy** (per research pitfall 4): widget drawer at z-50 so it stays above grid but below future person card (z-60) and modals (z-70).
