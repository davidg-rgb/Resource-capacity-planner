<objective>
Refactor the 5 legacy register services (`person.service.ts`, `project.service.ts`, `program.service.ts`, `department.service.ts`, `discipline.service.ts`) so their mutation functions (`create*`, `update*`, `delete*`/`archive*`) delegate to `register.service.ts`'s `createRegisterRow / updateRegisterRow / archiveRegisterRow` instead of calling `withTenant().insert/update/delete` directly. Thread `actorUserId` from `requireRole('admin')` through the 10 mutating route handlers. This closes the audit-spine gap on all five legacy register routes — AUDIT-01 through AUDIT-05.

Purpose: After v5.0 introduced `register.service.ts` as the single canonical writer that records a `change_log` row inside every mutation transaction (ADR-003), the v4 routes still wrote silently via `withTenant().insertPerson(...)` etc. — every mutation on those 5 routes bypassed the audit spine. This plan removes the dual write path by making the legacy services thin shims while preserving every route signature, response envelope, and call-site contract so consumers (`use-people.ts`, `use-projects.ts`, `use-reference-data.ts`, PersonForm, onboarding wizard) are unaffected.

Output: 5 rewritten service files + 10 updated route files. No new tests in this plan (54-03 covers contract tests). After this plan, every PUT/POST/DELETE on the 5 legacy register routes emits a `change_log` row.

Pattern decision: **Pattern 1 from RESEARCH.md** — service-shim delegation. Reason: smallest diff, easiest rollback, preserves all import paths (`createPerson` etc. continue to exist). Pattern 2 (route → register.service direct) is deferred to a v8 cleanup if desired.
</objective>
