<objective>
Add explicit RuleTester unit tests in `require-change-log.rule.test.ts` that lock in detection of the 5 mutating verbs added to `MUTATION_PREFIX_REGEX` in commit `e1d5e6d` (2026-05-10): `execute`, `promote`, `apply`, `cancel`, `stage`. Each verb gets a paired valid (with `recordChange`) + invalid (missing `recordChange`, no escape hatch) case. The regex itself ships already; this plan covers AUDIT-07 by preventing silent regression if a future edit narrows the regex.

Purpose: AUDIT-07 success criterion 3 ‚Äî "an eslint test confirms a sample mutating function with each new prefix fails the rule without `@no-change-log` or a `change_log` write." Without these tests, the regex could be narrowed by a future refactor and the build would stay green even as the rule lost coverage.

Output: Updated `src/features/change-log/__tests__/require-change-log.rule.test.ts` with 10 new RuleTester cases (5 valid + 5 invalid), all passing under `pnpm test`.
