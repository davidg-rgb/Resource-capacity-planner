    **Types** — `import-wizard.types.ts`:
    ```ts
    export type WizardStep = 'upload' | 'preview' | 'unmatched' | 'confirm' | 'result';
    export type NameResolution = { input: string; kind: 'person'|'project'; resolvedId: string | null; action: 'fuzzy-accept'|'manual-pick'|'mark-new' };
    export type WizardState = {
      step: WizardStep;
      sessionId: string | null;
      preview: PreviewResult | null;
      resolutions: NameResolution[];
      overrideManualEdits: boolean;
      overrideUnrolledImports: boolean;
      committedBatchId: string | null;
      committedAt: string | null;
      error: { code: string; message: string } | null;
      isLoading: boolean;
    };
    ```

    **Hook** — `use-import-wizard.ts`: a small `useReducer`-based hook (no new state lib). Exposes actions `uploadFile(file)`, `reloadPreview()`, `setResolution(res)`, `toggleOverrideManual()`, `toggleOverridePriorBatch()`, `commit()`, `rollback()`, `reset()`. Each action calls the corresponding `/api/v5/imports/*` endpoint via `fetch`, normalizes errors into `state.error`, advances `state.step` on success.

    Error routing:
      - `ERR_US_WEEK_HEADERS`, `ERR_UNKNOWN_LAYOUT`, `UNSUPPORTED_FILE_TYPE` → stay on `'upload'` step, populate `error`.
      - `PRIOR_BATCH_ACTIVE` → stay on `'confirm'` step, force-show overrideUnrolledImports checkbox.
      - `UNRESOLVED_NAMES` → revert to `'unmatched'` step.
      - `BATCH_ALREADY_ROLLED_BACK`, `ROLLBACK_WINDOW_EXPIRED` → show toast/message on `'result'`.

    - Three count badges (new / updated / warnings), each with `tabular-nums`.

    **UnmatchedNamesPanel.tsx** — Props: `{ unmatchedNames, resolutions, onResolve }`. For each unmatched entry render a row:
    - Input name + kind badge (person/project).
    - If `match.kind === 'fuzzy'`: show the single suggestion + confidence pill + "Accept" button → `onResolve({action:'fuzzy-accept', resolvedId: match.id})`.
    - If `match.kind === 'ambiguous'`: show a dropdown with the top candidates, user picks → manual-pick.
    - If `match.kind === 'none'`: show a searchable picker (reuse existing `SearchablePersonPicker`/`SearchableProjectPicker` if present; otherwise a plain `<select>` fed by a list fetched from an existing people/projects list endpoint — grep for the existing pattern before inventing).
    - "Mark as new" button disabled for v5.0 (WIZ-01 does not create people on the fly; the button exists but is disabled with a tooltip). NO functional mark-as-new in this plan — future work.
    - Disable the wizard's Next button until every unmatched row has a resolution.

    **RollbackButton.tsx** — Props: `{ batchId, committedAt }`. Computes `hoursRemaining = 24 - (now - committedAt in hours)`. If `<= 0` → render disabled with tooltip (`t('rollbackExpired')`). Otherwise clickable button that opens a confirm dialog → POST rollback → on success show success message + disable. On `ROLLBACK_WINDOW_EXPIRED` or `BATCH_ALREADY_ROLLED_BACK` → update state and disable. Shows `t('rollbackWindow', {hours: hoursRemaining.toFixed(1)})`.

    **ImportWizard.tsx** — The top-level component. Uses `useImportWizard` hook. Switch on `state.step`:
      - `'upload'`: `<ImportDropzone onFile={uploadFile}/>`, show parse-error alerts.
      - `'preview'`: `<ImportPreviewTable preview={state.preview!}/>` + override checkboxes + Back/Next.
      - `'unmatched'`: `<UnmatchedNamesPanel .../>` + Back/Next (Next disabled until resolved).
      - `'confirm'`: Summary + final commit button, handles `PRIOR_BATCH_ACTIVE` by surfacing the second override.
      - `'result'`: Success summary + `<RollbackButton batchId={state.committedBatchId!} committedAt={state.committedAt!}/>` + "Import another" button → `reset()`.

    All strings via `useTranslations('v5.import')`. No JSX text literals.
