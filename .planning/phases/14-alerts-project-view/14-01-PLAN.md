       ```typescript
       // --- Alert types (Phase 14) ---
       export type AlertSeverity = 'overloaded' | 'underutilized';

       export interface CapacityAlert {
         personId: string;
         firstName: string;
         lastName: string;
         departmentName: string;
         totalTarget: number;
         totalAllocated: number;
         utilizationRatio: number;   // e.g. 1.25 = 125%, 0.3 = 30%
         severity: AlertSeverity;
       }

       // --- Project staffing types (Phase 14) ---
       export interface ProjectStaffingPerson {
         personId: string;
         firstName: string;
         lastName: string;
         targetHoursPerMonth: number;
         months: Record<string, number>; // YYYY-MM -> hours allocated to this project
       }

       export interface ProjectStaffingResponse {
         projectId: string;
         projectName: string;
         people: ProjectStaffingPerson[];
         months: string[];       // ordered YYYY-MM array
         generatedAt: string;
       }
       ```

    2. **Extend analytics.service.ts** -- add three new exported functions after existing ones:

       a. `getCapacityAlerts(orgId, monthFrom, monthTo): Promise<CapacityAlert[]>` -- Same CTE pattern as `getDashboardKPIs` but returns individual person rows where `utilization_ratio > 1.0 OR utilization_ratio < 0.5`. Include person_id, first_name, last_name, department_name (INNER JOIN departments), total_target, total_allocated, utilization_ratio. Map severity based on ratio. Use `CASE WHEN total_target > 0` for division-by-zero guard. ORDER BY utilization_ratio DESC.

       b. `getAlertCount(orgId, monthFrom, monthTo): Promise<number>` -- Same CTE but only `SELECT COUNT(*)::int AS count FROM person_utilization WHERE utilization_ratio > 1.0 OR utilization_ratio < 0.5`. Returns just the count number.

       c. `getProjectStaffing(orgId, projectId, monthFrom, monthTo): Promise<ProjectStaffingResponse>` -- Uses generate_series CTE for gapless month grid. INNER JOIN people (WHERE archived_at IS NULL) with allocations filtered by projectId. Returns per-person-per-month hours. Also query project name. Use the pattern from 14-RESEARCH.md Pattern 2. Import `generateMonthRange` from date-utils for the months array. The SQL should GROUP BY person fields + month, LEFT JOIN allocations on person_id + project_id + month.

       Import the new types: `CapacityAlert, ProjectStaffingPerson, ProjectStaffingResponse`.

    3. **Create API routes** following the pattern in `src/app/api/analytics/dashboard/route.ts`:

       a. `src/app/api/analytics/alerts/route.ts` -- GET handler. Parse `from` and `to` query params, validate with MONTH_RE. Call `getCapacityAlerts(orgId, from, to)`. Add feature flag check: import `getOrgFlags` from `@/features/flags/flag.service` and `getTenantId` from `@/lib/auth`. After getting orgId, call `getOrgFlags(orgId)` and check `flags.alerts`. If false, return 403 `{ error: 'Feature not enabled' }`.

       b. `src/app/api/analytics/alerts/count/route.ts` -- GET handler. Same pattern but calls `getAlertCount`. Same flag check.

       c. `src/app/api/analytics/project-staffing/route.ts` -- GET handler. Parse `projectId`, `from`, `to` query params. Validate projectId is present and from/to match MONTH_RE. Call `getProjectStaffing(orgId, projectId, from, to)`. No flag check needed (project view is not separately flagged).

    4. **Create hooks:**

       a. `src/hooks/use-alerts.ts`:
          - `useAlerts(monthFrom, monthTo)` -- queryKey `['alerts', monthFrom, monthTo]`, staleTime 60_000, fetches `/api/analytics/alerts?from=${monthFrom}&to=${monthTo}`, returns `UseQueryResult<CapacityAlert[]>`.
          - `useAlertCount(monthFrom, monthTo)` -- queryKey `['alert-count', monthFrom, monthTo]`, staleTime 30_000 (shorter for badge freshness), `refetchOnWindowFocus: true`, fetches `/api/analytics/alerts/count?from=${monthFrom}&to=${monthTo}`, returns `UseQueryResult<number>`.

       b. `src/hooks/use-project-staffing.ts`:
          - `useProjectStaffing(projectId, monthFrom, monthTo)` -- queryKey `['project-staffing', projectId, monthFrom, monthTo]`, staleTime 60_000, `enabled: !!projectId`, fetches `/api/analytics/project-staffing?projectId=${projectId}&from=${monthFrom}&to=${monthTo}`, returns `UseQueryResult<ProjectStaffingResponse>`.

    Also add `'alert-count'` and `'alerts'` invalidation in `src/hooks/use-grid-autosave.ts` alongside the existing `['allocations', personId]` invalidation in all three places (line 85, 89, 130/133). This ensures alert counts refresh after allocation edits.
