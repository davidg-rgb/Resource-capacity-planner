       export type AnnouncementSeverity = 'info' | 'warning' | 'critical';

       export interface Announcement {
         id: string;
         title: string;
         body: string;
         severity: AnnouncementSeverity;
         targetOrgIds: string[] | null;
         startsAt: string;
         expiresAt: string | null;
         createdAt: string;
       }

    2. **Zod schemas** (src/features/announcements/announcement.schema.ts):
       - `createAnnouncementSchema`: title (string, 1-200), body (string, 1-2000), severity (enum: info|warning|critical), targetOrgIds (optional array of UUID strings), startsAt (ISO date string), expiresAt (optional ISO date string)
       - `updateAnnouncementSchema`: Same fields but all optional (partial update)

    3. **Service** (src/features/announcements/announcement.service.ts):
       - `getActiveAnnouncements(orgId?: string)`: Query systemAnnouncements where `startsAt <= NOW()` AND `(expiresAt IS NULL OR expiresAt > NOW())` AND (targetOrgIds is NULL OR contains orgId). Order by severity (critical first). MUST filter dates server-side per Pitfall 4 in research.
       - `listAnnouncements()`: Return all announcements ordered by createdAt desc (for platform admin listing).
       - `createAnnouncement(data, adminId)`: Insert into systemAnnouncements with createdByAdminId = adminId.
       - `updateAnnouncement(id, data)`: Partial update of announcement.
       - `deleteAnnouncement(id)`: Delete announcement by ID.

    4. **Platform API: GET + POST /api/platform/announcements** (route.ts):
       - GET: Call `requirePlatformAdmin()`, then `listAnnouncements()`, return JSON array.
       - POST: Call `requirePlatformAdmin()`, validate body with createAnnouncementSchema, call `createAnnouncement(data, adminId)`, return 201 with created object.
       - Follow existing platform API route patterns (check src/app/api/platform/tenants/route.ts for auth pattern).

    5. **Platform API: PATCH + DELETE /api/platform/announcements/[id]** ([id]/route.ts):
       - PATCH: `requirePlatformAdmin()`, validate body with updateAnnouncementSchema, call `updateAnnouncement(id, data)`, return updated object.
       - DELETE: `requirePlatformAdmin()`, call `deleteAnnouncement(id)`, return 204.

    6. **Tenant API: GET /api/announcements/active** (route.ts):
       - Call `getTenantId()` to get current org's internal UUID
       - Call `getActiveAnnouncements(orgId)` -- pass orgId so the query filters by targetOrgIds
       - Return JSON array of active announcements
       - This endpoint is used by the tenant-facing banner component
