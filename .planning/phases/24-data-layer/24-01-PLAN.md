Add 3 new exported async functions to `analytics.service.ts`. All follow the existing CTE + generate_series pattern established by `getTeamHeatMap`.

**1. `getCapacityForecast(orgId, monthFrom, monthTo, filters?: { projectId?, departmentId? }): Promise<CapacityForecastResponse>`**

SQL structure (single query):
- `month_series` CTE: `generate_series(fromDate, toDate, '1 month')` formatted as YYYY-MM
- `active_people` CTE: people WHERE organization_id = orgId AND archived_at IS NULL, with optional department filter (same pattern as getTeamHeatMap deptFilter)
- `supply` CTE: `SELECT ms.month, SUM(ap.target_hours)::int AS total FROM active_people ap CROSS JOIN month_series ms GROUP BY ms.month`
- `demand` CTE: `SELECT to_char(a.month, 'YYYY-MM') AS month, SUM(a.hours)::int AS total FROM allocations a INNER JOIN active_people ap ON ap.id = a.person_id WHERE a.organization_id = orgId AND month in range GROUP BY month`. If projectId filter provided, add `AND a.project_id = projectId::uuid`.
- Final SELECT: LEFT JOIN supply and demand onto month_series, COALESCE to 0.

Transform rows into response: build `months[]`, `supply` Record, `demand` Record, `gap` Record (supply - demand). For summary: surplusMonths = gap > 5% of supply, deficitMonths = gap < -5% of supply, balancedMonths = rest. Add generatedAt.

**2. `getCapacityDistribution(orgId, monthFrom, monthTo, groupBy: 'project' | 'department' | 'discipline', limit = 8): Promise<CapacityDistributionResponse>`**

SQL structure:
- `month_series` + `active_people` CTEs (same as above)
- Main query: JOIN allocations to active_people, GROUP BY the grouping dimension + month. The groupBy determines which table to join and which id/name to select:
  - `project`: JOIN projects pr ON pr.id = a.project_id, group by pr.id, pr.name
  - `department`: group by ap.department_id, ap.department_name (already in active_people CTE which should JOIN departments)
  - `discipline`: JOIN disciplines disc ON disc.id = ap.discipline_id, group by disc.id, disc.name
- Also compute supply per month (same as capacity-forecast supply CTE).

Transform: Sort groups by totalHours DESC. Take top `limit` groups. If remaining groups exist, aggregate into `other` bucket. Assign colors from a fixed palette (define `const CHART_PALETTE = ['#3b82f6', '#10b981', '#f59e0b', '#ef4444', '#8b5cf6', '#ec4899', '#06b6d4', '#84cc16', '#f97316', '#6366f1']`). Generate insight string if largest group > 50% of total: `"${name} accounts for ${pct}% of allocated hours"`.

**3. `getDisciplineDemand(orgId, monthFrom, monthTo, filters?: { departmentId? }): Promise<DisciplineDemandResponse>`**

SQL structure:
- `month_series` + standard CTEs
- Per-discipline, per-month: supply = count of active people with that discipline * target_hours, demand = sum of allocation hours for people with that discipline.
- Use CROSS JOIN between disciplines and month_series to ensure gapless grid.
- LEFT JOIN people and allocations.
- GROUP BY discipline_id, discipline_name, abbreviation, month.

Transform: Build per-discipline months Record with demand, supply, gap, status (surplus if gap > 10% of supply, balanced if within +/-10%, tight if within -10% to -25%, deficit if gap < -25% of supply). Calculate peakDeficit/peakDeficitMonth. Use the `hasSustainedDeficit` pattern from research (3+ consecutive deficit months). Summary: combinedPeakDeficit = sum of all discipline peak deficits, fteHiringNeed = combinedPeakDeficit / average target_hours_per_month.

**1. `src/hooks/use-capacity-forecast.ts`**
```typescript
'use client';
import { useQuery, type UseQueryResult } from '@tanstack/react-query';
import type { CapacityForecastResponse } from '@/features/analytics/analytics.types';

export function useCapacityForecast(
  monthFrom: string,
  monthTo: string,
  filters?: { projectId?: string; departmentId?: string },
): UseQueryResult<CapacityForecastResponse> {
  return useQuery<CapacityForecastResponse>({
    queryKey: ['capacity-forecast', monthFrom, monthTo, filters],
    queryFn: async () => {
      const params = new URLSearchParams({ from: monthFrom, to: monthTo });
      if (filters?.projectId) params.set('projectId', filters.projectId);
      if (filters?.departmentId) params.set('departmentId', filters.departmentId);
      const response = await fetch(`/api/analytics/capacity-forecast?${params}`);
      if (!response.ok) throw new Error(`Failed to fetch capacity forecast: ${response.status}`);
      return response.json();
    },
    staleTime: 60_000,
  });
}
```
