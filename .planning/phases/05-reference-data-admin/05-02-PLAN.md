**Discipline hooks:**

`useDisciplines()` -- queryKey `['disciplines']`, GET /api/disciplines, return data.disciplines as DisciplineRow[]

`useDiscipline(id: string)` -- queryKey `['disciplines', id]`, GET /api/disciplines/{id}, return `{ discipline: DisciplineRow, usageCount: number }`. Enable only when id is truthy.

`useCreateDiscipline()` -- POST /api/disciplines, body `{ name, abbreviation }`, onSuccess invalidate `['disciplines']`

`useUpdateDiscipline()` -- PATCH /api/disciplines/{id}, body partial `{ name?, abbreviation? }`, onSuccess invalidate `['disciplines']` and `['disciplines', id]`

`useDeleteDiscipline()` -- DELETE /api/disciplines/{id}, onSuccess invalidate `['disciplines']` and `['people']` (people reference disciplines)

**Department hooks** -- Same pattern, queryKey `['departments']`:

`useDepartments()`, `useDepartment(id)`, `useCreateDepartment()`, `useUpdateDepartment()`, `useDeleteDepartment()`

Delete invalidates `['departments']` and `['people']`.

**Program hooks** -- Same pattern, queryKey `['programs']`:

`usePrograms()`, `useProgram(id)`, `useCreateProgram()`, `useUpdateProgram()`, `useDeleteProgram()`

Delete invalidates `['programs']` and `['projects']`.

All mutation hooks follow the exact pattern from use-people.ts: useMutation with mutationFn that throws on !res.ok, onSuccess that invalidates queryClient.
