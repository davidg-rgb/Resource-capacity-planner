# Phase 3 — UI Design Contract

## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element spacing |
| sm | 8px | Compact element spacing, nav item vertical padding |
| md | 16px | Default element spacing, sidebar item horizontal padding |
| lg | 24px | Section padding, sidebar section padding (px-6) |
| xl | 32px | Layout gaps, main content padding (p-8) |
| 2xl | 48px | Major section breaks, sidebar header area (py-8 in prototype-06) |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Top nav height: 57px (fixed, derived from py-3 + content height per prototype-08)
- Sidebar width: 256px (w-64, consistent across prototypes 01/06)
- Sidebar width (person list variant): 288px (w-72, prototype-08)
- Main content max-width: 1440px (mx-auto max-w-[1440px])

Source: All spacing values extracted from prototypes 01, 06, 08 which share identical shell chrome dimensions.

## Color

All color tokens already declared in `src/app/globals.css` @theme block. Phase 3 uses existing tokens without modification.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-surface` | #fafcff | Page background, main content area |
| Secondary (30%) | `--color-surface-container-low` | #f3f5fa | Sidebar background, search input bg |
| | `--color-surface-container` | #eef1f6 | Card backgrounds, hover states |
| | `--color-surface-container-high` | #e8eaf0 | Active sidebar item background, input backgrounds |
| Accent (10%) | `--color-primary` | #496173 | See reserved-for list below |
| Destructive | `--color-error` | #ba1a1a | Error states, destructive actions only |
| Text primary | `--color-on-surface` | #1a1c1e | Headings, body text |
| Text secondary | `--color-on-surface-variant` | #43474e | Descriptions, subtitles |
| Text tertiary | `--color-outline` | #73777f | Placeholder text, metadata labels |
| Border | `--color-outline-variant` | #c3c7cf | Dividers, card borders (typically at /10 or /15 opacity) |

### Accent Reserved For

The primary accent `#496173` is reserved exclusively for:
- Active top nav item text + bottom border indicator (2px)
- App title text ("Nordic Capacity")
- Primary CTA buttons (bg-primary with on-primary text)
- Active sidebar item text color
- Sidebar logo icon background
- Focus ring on inputs (ring-primary)

Accent must NOT be used for: general text, borders, background fills on non-interactive elements, or decorative purposes.

### Prototype-to-Token Alignment

The prototypes use a slightly different Material Design 3 palette (e.g. `surface: #f8fafb` vs globals.css `#fafcff`). The globals.css tokens are the canonical source of truth for implementation. The prototypes are visual reference only -- use the exact hex values from globals.css, not from prototype HTML.

## Interaction Contracts

### Route Protection

| State | Behavior |
|-------|----------|
| Unauthenticated + /app/* route | Redirect to /sign-in (silent, no flash of content) |
| Authenticated + no org | Redirect to /onboarding |
| Authenticated + has org | Render app shell with navigation |
| Insufficient role + API call | JSON 403 response, no UI toast (consuming code handles) |

### Navigation

| Interaction | Behavior |
|-------------|----------|
| Click top nav item | Navigate to section root, update active indicator |
| Click side nav item | Navigate within section, update active highlight |
| Browser back/forward | Standard Next.js App Router behavior |
| Active state detection | Derived from current pathname prefix matching |

### Auth Events

| Event | Feedback |
|-------|----------|
| Sign-in success | Redirect to /input (NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL) |
| Sign-up success | Redirect to /onboarding (NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL) |
| Sign-out | Redirect to / (landing page) |
| Session expired | Clerk handles automatic redirect to /sign-in |
| Invite accepted | Clerk handles, user lands on /input |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| >= 1024px (default) | Full app shell: top nav + sidebar + main content |
| < 1024px (md) | Top nav items hidden (future: hamburger menu -- not Phase 3 scope) |
| Minimum supported | 1024px (desktop-first, per scope doc) |

Phase 3 does not implement mobile/tablet breakpoints. The app requires minimum 1024px viewport width per project scope ("Desktop-first, responsive to 1024px" is a v2 consideration; MVP is desktop-only).
