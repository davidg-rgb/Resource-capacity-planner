1. **drizzle/migrations/0009_lean_trim_dead_widgets.sql** -- Create the one-shot migration. Per D-02, this MUST run before any widget file deletion. The SQL strips dead widget IDs from the `dashboard_layouts.layout` JSONB column. Use `COALESCE` to handle the edge case where all widgets in a row are dead (prevents null layout):

```sql
-- Phase 51: Strip dead widget IDs from custom dashboard layouts (LEAN-05)
-- Dead widgets: discipline-progress, discipline-demand, project-impact
-- These have 0 default layout placements but may exist in tenant custom layouts.
-- VERIFY-05 found 1 affected row on dev Neon branch.

UPDATE dashboard_layouts
SET layout = COALESCE(
  (
    SELECT jsonb_agg(placement)
    FROM jsonb_array_elements(layout) placement
    WHERE placement->>'widgetId' NOT IN (
      'discipline-progress', 'discipline-demand', 'project-impact'
    )
  ),
  '[]'::jsonb
)
WHERE layout::text ~* 'discipline-progress|discipline-demand|project-impact';
```

## Trust Boundaries

| dashboard_layouts JSONB -> SQL migration | Migration modifies tenant data in the database |
| widget ID strings -> widget-registry lookup | Layout arrays reference widgets by string ID |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-51-04 | Tampering | 0009_lean_trim_dead_widgets.sql | mitigate | All widget IDs in the WHERE clause are hardcoded string literals (`'discipline-progress'`, `'discipline-demand'`, `'project-impact'`) -- no user input, no interpolation. The COALESCE handles null edge case. Migration is idempotent (re-running on already-cleaned rows is a no-op). |
| T-51-05 | Denial of Service | JSONB migration on large table | accept | `dashboard_layouts` is a small table (< 100 rows per tenant, typically < 10). The regex match in WHERE clause limits UPDATE to only affected rows. No performance risk. |
| T-51-06 | Elevation of Privilege | Feature flag bypass | accept | `uiV6LeanTrim` flag controls layout selection only (cosmetic). Not a security boundary per ADR-004. Flag bypass would show trimmed/original layouts -- no privilege escalation. |
</threat_model>
