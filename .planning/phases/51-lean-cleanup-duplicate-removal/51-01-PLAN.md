## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External URL -> next.config.ts redirects | User-typed URLs enter the redirect system |
| dashboard_layouts JSONB -> widget-registry | Tenant-stored widget IDs are looked up in the registry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-51-01 | Spoofing | next.config.ts redirects | mitigate | All redirect destinations are hardcoded strings (`/admin/people`, `/admin/projects`, `/pm/wishes`) -- no user input in destination. The `/team/:path*` wildcard maps only to `/admin/people/:path*` (same-origin, hardcoded prefix). No open-redirect risk. |
| T-51-02 | Information Disclosure | widget fallback placeholder | accept | Placeholder shows widget ID string (e.g. `discipline-progress`). This is an internal identifier, not sensitive data. Visible only to authenticated dashboard users. |
| T-51-03 | Denial of Service | widget fallback rendering | accept | A layout with many unknown widget IDs renders many placeholder divs. Bounded by layout array length (typically <20 widgets). No amplification risk. |
</threat_model>
