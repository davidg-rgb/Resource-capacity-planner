# Phase 25 Plan 01: Gauges + Sparklines (C1)

## Objective

Build the two simplest visualization components (V6 Department Capacity Gauges and V4 Utilization Trend Sparklines) as the first smoke test of the dashboard widget framework + data layer.

## Tasks

1. **V6 Department Capacity Gauges** — Radial 270-degree donut gauge per department using Recharts PieChart. Color-coded (green/amber/red), center percentage label, headcount, trend arrow, click-to-navigate.

2. **V4 Utilization Trend Sparklines** — Custom SVG sparkline (80x24px path) per department/person in a table layout. View toggle (departments vs top-10 people), direction arrows, overload warnings.

3. **Widget Wrappers** — Wire both components to `useUtilizationTrends` hook via widget wrapper pattern. Handle loading, error, and empty states.

4. **Registry Registration** — Register both as `health-capacity` category widgets on the `manager` dashboard. Add side-effect imports to `widgets/index.ts`.

## Success Criteria

- Both widgets render with proper loading/error/empty states
- Both implement `WidgetProps` interface
- Both registered in widget registry
- TypeScript compiles without new errors
- Gauges use Recharts PieChart with 270-degree arc
- Sparklines use custom SVG path (no Recharts dependency)
