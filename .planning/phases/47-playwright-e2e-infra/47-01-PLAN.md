Create `scripts/generate-e2e-import-fixture.ts` — a pure, deterministic generator that writes `e2e/fixtures/nordlys-import.xlsx` using the `xlsx` package already in dependencies.

Content requirements:
- 3 people from the deterministic seed: Anna, Per, Sara (use their full names as appearing in `tests/fixtures/seed.ts` PEOPLE table — Anna Lindqvist, Per Karlsson, Sara Berg)
- 3 months: 2026-05, 2026-06, 2026-07
- Project: Nordlys
- Hours per row: Anna 80/60/80, Per 40/40/40, Sara 60/60/60 (distinct enough that the preview UI shows variety; 60/2026-06/Sara matches the seeded rejected-proposal cell for pleasant narrative continuity but is NOT required)
- Column headers matching the existing import template schema — read `scripts/generate-import-template.ts` to copy the exact header row format, so the test import hits the happy path (no "unknown column" warnings).

The generator script:
1. Imports `XLSX` from `xlsx` package
2. Builds a worksheet array-of-arrays (header row + 9 data rows)
3. `XLSX.utils.aoa_to_sheet()` → `XLSX.utils.book_new()` → `XLSX.writeFile(..., 'e2e/fixtures/nordlys-import.xlsx')`
4. Logs a success line

Run the script once to produce the file: `pnpm tsx scripts/generate-e2e-import-fixture.ts`. Commit BOTH the script and the resulting xlsx.

Rationale for committing the xlsx directly (not regenerating at test time): binary fixture is small, deterministic, and removes a test-time dependency on tsx availability in CI. The script is kept so a maintainer can regenerate if the template schema ever changes.
