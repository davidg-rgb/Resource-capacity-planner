    1. Install dependencies: `pnpm add jose bcryptjs` and `pnpm add -D @types/bcryptjs`.

       - Import `jwtVerify`, `SignJWT` from `jose`, `cookies` from `next/headers`.
       - Import `AuthError` from `./errors`.
       - Export const `PLATFORM_COOKIE = 'platform-token'`.
       - Export interface `PlatformAdmin { adminId: string; email: string; name: string; }`.
       - `getSecret()` helper: `new TextEncoder().encode(env.PLATFORM_ADMIN_SECRET)`. If `PLATFORM_ADMIN_SECRET` is undefined, throw `AuthError('Platform admin not configured')`.
       - Export `signPlatformToken(payload: PlatformAdmin): Promise<string>` using `new SignJWT({...payload}).setProtectedHeader({alg:'HS256'}).setIssuedAt().setExpirationTime(env.PLATFORM_ADMIN_TOKEN_EXPIRY).setIssuer('nordic-capacity-platform').sign(getSecret())`.
       - Export `requirePlatformAdmin(): Promise<PlatformAdmin>` — reads `platform-token` cookie, verifies with `jwtVerify(token, getSecret(), { issuer: 'nordic-capacity-platform' })`, returns PlatformAdmin from payload. Throws `AuthError('Platform authentication required')` if no cookie, `AuthError('Invalid or expired platform token')` on verify failure.

    3. Create `src/lib/platform-audit.ts` (per D-18):
       - Import `db` from `@/db`, `platformAuditLog` from `@/db/schema`, `headers` from `next/headers`.
       - Export interface `AuditEntry { adminId: string; action: string; targetOrgId?: string; targetUserId?: string; impersonationSessionId?: string; details?: Record<string, unknown>; }`.
       - Export `logPlatformAction(entry: AuditEntry): Promise<void>` — gets IP from `x-forwarded-for` (first value) then `x-real-ip` then `'unknown'`. Gets userAgent from `user-agent` header. Inserts into `platformAuditLog`.

    4. Create `src/features/platform/platform-auth.schema.ts`:
       - Export `loginSchema = z.object({ email: z.email(), password: z.string().min(1) })`.

    5. Create `src/features/platform/platform-auth.service.ts` (per D-01):
       - Import `bcrypt` from `bcryptjs`, `eq` from `drizzle-orm`, `db` from `@/db`, `platformAdmins` from `@/db/schema`, `AuthError` from `@/lib/errors`.
       - Export `verifyPlatformLogin(email: string, password: string)` — selects from platformAdmins by email, checks isActive, compares password with bcrypt.compare, updates lastLoginAt, returns `{ id, email, name }`. Throws `AuthError('Invalid credentials')` on failure.

    6. Update `src/proxy.ts` (per D-03, D-04, PLAT-10):
       - Add `'/platform(.*)'` and `'/api/platform(.*)'` to the `isPublicRoute` matcher array. This ensures Clerk does NOT intercept platform routes.

       - Change `PLATFORM_ADMIN_SECRET` from `.optional()` to required: `z.string().min(64)`. Platform admin is now a real feature, not optional. All environments must have it set.

       - Call `logPlatformAction({ adminId: admin.adminId, action: 'admin.logout' })`.
       - Delete the `platform-token` cookie (set maxAge=0, same path/domain).
       - Return `{ success: true }` with 200.

    3. Create `src/app/api/platform/auth/me/route.ts`:
       - GET handler: call `requirePlatformAdmin()`, return the admin identity as JSON.
       - On error, return handleApiError.

       - Read `process.env.PLATFORM_ADMIN_EMAIL` and `process.env.PLATFORM_ADMIN_PASSWORD` directly (NOT from env.ts — seed runs outside Next.js).
