1. Create `src/components/import/step-import.tsx`:
  - Props: `{ importStatus: 'idle' | 'importing' | 'success' | 'error'; importResult: ImportResult | null; onExecute: () => void }`
  - Three states:
    a. **idle**: Show summary of what will be imported: "Ready to import X allocations". "Import" primary button. Per D-03: this is the final confirmation before executing.
    b. **importing**: Per D-03: Show progress indicator — a spinner/loading animation with text "Importing allocations... Please don't close this page." Disable all navigation (back button, stepper clicks).
    c. **success**: Per D-03: Show results summary card:
      - "Import Complete" heading with green checkmark
      - "{X} rows imported" in large text
      - "{Y} skipped" if any
      - "{Z} warnings" if any, listed below
      - "Done" button (primary) that navigates to `/data` using `useRouter().push('/data')`
    d. **error**: Show error message from `importResult.error` with red styling. "The import was rolled back. No data was changed." reassurance text. "Try Again" button that resets to idle. "Back" button to return to validate step.

2. Update `src/components/import/import-wizard.tsx` — replace placeholders with real step components:

  - Import all step components: StepUpload, StepMap, StepValidate, StepImport.
  - Import hooks: useValidateRows, useExecuteImport from `@/hooks/use-import`.
  - Import utilities: unpivotData from `@/features/import/import.utils`.

  **State management:**
  - Full WizardState managed with useState. Initialize with step='upload', everything else null/empty.

  **Step handlers:**
  - `handleFileUploaded(parsedFile, mappings)`: Set parsedFile and columnMappings in state, advance to 'map'.
  - `handleMappingsConfirmed()`: Advance to 'validate'. Trigger validation:
    - If formatInfo.isPivot: call `unpivotData(parsedFile.allRows, parsedFile.headers, parsedFile.formatInfo, columnMappings)` to get flat ImportRow[].
    - If flat format: map allRows to ImportRow[] using columnMappings (for each row, extract values at mapped column indices, construct ImportRow with rowIndex).
    - Call `validateMutation.mutateAsync({ rows: importRows })`. On success, set validationResult in state.
  - `handleUserFixesChange(fixes)`: Update userFixes in state.
  - `handleValidationConfirmed()`: Advance to 'import' step. Set importStatus to 'idle'.
  - `handleExecuteImport()`: Set importStatus to 'importing'. Build the execute payload:
    - For each validation row with status 'ready' or 'warning':
      - personId = userFixes[row.rowIndex]?.personId ?? row.personMatch.matchId (must exist for ready/warning rows)
      - projectId = userFixes[row.rowIndex]?.projectId ?? row.projectMatch.matchId
      - hours = userFixes[row.rowIndex]?.hours ?? row.data.hours
      - month = row.data.month
    - Call `executeMutation.mutateAsync({ rows })`.
    - On success: set importStatus='success', importResult.
    - On error: set importStatus='error', importResult with error message.

  **Step rendering (replace placeholder divs):**
  - 'upload': `<StepUpload onFileUploaded={handleFileUploaded} />`
  - 'map': `<StepMap headers={state.parsedFile.headers} sampleRows={state.parsedFile.sampleRows} mappings={state.columnMappings} onMappingsChange={(m) => setState(s => ({...s, columnMappings: m}))} onNext={handleMappingsConfirmed} />`
  - 'validate': `<StepValidate validationResult={state.validationResult} userFixes={state.userFixes} onUserFixesChange={handleUserFixesChange} onNext={handleValidationConfirmed} />` (show loading spinner while validateMutation.isPending)
  - 'import': `<StepImport importStatus={state.importStatus} importResult={state.importResult} onExecute={handleExecuteImport} />`

  **Back button logic (per D-02):**
  - 'map' -> 'upload': keep parsedFile in state (preserves upload data)
  - 'validate' -> 'map': keep columnMappings in state, clear validationResult
  - 'import' -> 'validate': only if importStatus is 'idle' or 'error'. Keep validationResult.
  - Disable back during 'importing' status.

  **Loading states:**
  - While validateMutation.isPending: show skeleton/spinner over validate step content
  - While executeMutation is running: handled by step-import component's 'importing' state
