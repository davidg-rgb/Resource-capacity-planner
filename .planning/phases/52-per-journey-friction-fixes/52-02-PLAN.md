Service function signature (append to `src/features/proposals/proposal.service.ts`):
```ts
export async function getQueueCount(orgId: string, departmentId: string): Promise<number> {
  const [row] = await db
    .select({ count: sql<number>`count(*)::int` })
    .from(schema.allocationProposals)
    .innerJoin(schema.people, eq(schema.allocationProposals.personId, schema.people.id))
    .where(
      and(
        eq(schema.allocationProposals.organizationId, orgId),
        eq(schema.allocationProposals.status, 'proposed'),
        eq(schema.people.departmentId, departmentId),
      ),
    );
  return Number(row?.count ?? 0);
}
```

Route handler skeleton (`src/app/api/v5/proposals/queue/count/route.ts`):
```ts
import { NextRequest, NextResponse } from 'next/server';
import { z } from 'zod';
import { getQueueCount } from '@/features/proposals/proposal.service';
import { handleApiError } from '@/lib/api-utils';
import { requireRole } from '@/lib/auth';

const Query = z.object({ departmentId: z.string().uuid() });

export async function GET(request: NextRequest) {
  try {
    const { orgId } = await requireRole('planner');
    const parsed = Query.parse(Object.fromEntries(request.nextUrl.searchParams));
    const count = await getQueueCount(orgId, parsed.departmentId);
    return NextResponse.json({ count, departmentId: parsed.departmentId }, { status: 200 });
  } catch (error) {
    return handleApiError(error);
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API | Untrusted HTTP query string crosses here; request parser must validate `departmentId` as UUID before passing to SQL |
| API route → service | `orgId` is derived from Clerk auth (trusted); `departmentId` is parsed (untrusted until zod-validated) |
| service → DB | All DB access is scoped by `organizationId` in the WHERE clause; cross-tenant `departmentId` returns 0 rows naturally |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-01 | Spoofing | Route handler | mitigate | `requireRole('planner')` (Clerk middleware) rejects unauthenticated / wrong-role requests → 401/403 |
| T-52-02 | Tampering | Query-string `departmentId` | mitigate | Zod `z.string().uuid()` rejects non-UUIDs before the query runs → 400 |
| T-52-03 | Repudiation | — | accept | Read-only count endpoint; no state change to audit; change_log not required |
| T-52-04 | Information Disclosure | `getQueueCount` SQL | mitigate | WHERE clause filters on `organizationId = orgId`; cross-tenant dept id returns `count=0`. Explicit integration test `tenant isolation` asserts this |
| T-52-05 | Information Disclosure | Error logs | mitigate | `handleApiError` must NOT log `departmentId` values in plaintext beyond the zod issue message; no PII in this endpoint (UUIDs only) |
| T-52-06 | Denial of Service | DB count query | mitigate | Indexed lookup on `(organization_id, status)` + `people.department_id`; worst case O(n) scan on single-org dataset (acceptable — LM-01 polls once per minute per user) |
| T-52-07 | Elevation of Privilege | — | accept | Endpoint only requires `'planner'` role (which line-manager personas hold); no privilege escalation path |
</threat_model>
