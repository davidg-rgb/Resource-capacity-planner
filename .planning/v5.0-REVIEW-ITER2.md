## Yellow — Doc Drift (update architecture doc in next pass)

These are intentional implementation decisions that diverge from the spec text:

- `proposed_hours` is `numeric(5,2)` not `integer` (spec §7.1 line 1142)
- `import_status` enum has `'committed'` not `'cancelled'` (spec §7.5)
- Route prefix `/api/v5/proposals/` vs spec's `/api/v5/planning/proposals/`
- Withdraw uses `POST .../withdraw` not `DELETE .../[id]`
- Admin landing `/admin` not `/admin/change-log` (Phase 43 consolidation)
- Approval queue at `/approval-queue` not `/approvals`
- Admin registers at `/admin/people` not `/admin/registers/people`
- R&D drilldown is inline modal not separate `/rd/drilldown` route
- Function names: `getISOWeek`/`getISOWeekYear` vs `isoWeek()`, `matchPersonName` vs `matchPersonByName`, etc.
- `classifyHeader` in parser not iso-calendar (colocation decision)
- CellView type diverges (plannedHours vs approvedHours, single pendingProposal vs array)
- TC-CAL-011/027 spec contradiction: week 53 of 2026 = 3 working days (not 4 or 5)
- `getPersonaScopeFilter` not implemented as named function (done ad-hoc at page level)
- `withChangeLog` helper not implemented (spec says opt-in)
- Reversal payload missing `version:1` field
- Error code `SESSION_ALREADY_COMMITTED` vs spec's `SESSION_NOT_STAGED`
- ParsedRow missing `grain`, `isoWeek`, `monthKey` fields
