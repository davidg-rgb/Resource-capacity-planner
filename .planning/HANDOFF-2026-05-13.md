## Known landmines for the executor

From RESEARCH §6:
1. **Response envelope drift** — legacy routes wrap responses as `{ person }` / `{ project }` / etc.; the v5 admin route uses `{ row }`. Must preserve legacy envelopes byte-for-byte.
2. **Dependent-row 409 shape change** — legacy routes return `{ usageCount }`; register.service returns `{ entity, id, blockers }`. Plan 54-02 calls this out — decide on shape during execution (current plan keeps legacy envelope and translates).
3. **Project archive semantics** — register.service uses `status='archived' AND archivedAt != null`; legacy uses `archivedAt != null` alone. Plan 54-02 will follow register.service.
4. **PATCH un-archive no-op** — passing `archivedAt: null` on the legacy PATCH currently un-archives; new flow strips it via Zod silently. Plan 54-03 asserts this behavior in a dedicated test.
5. **Lint rule on shims** — the eslint rule looks for `recordChange(` calls in the function body; shim bodies call `createRegisterRow` not `recordChange`. Plan 54-03 step 4 extends the coverage test to recognize the dispatcher rather than adding `@no-change-log` escape hatches.
6. **Regex pre-existing violations** — non-risk: regex change already landed 3 days ago without breaking the build, so no latent violations exist.
