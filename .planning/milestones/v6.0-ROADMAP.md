## Overview

v6.0 was a pure restructure milestone — no new features. Goal: every user journey at its target click-count, persona-blind chrome removed, duplicate surfaces deleted, broken Line Manager / PM / Admin pages unblocked. All work was gated behind 4 independent feature flags (`uiV6.landing`, `uiV6.leanTrim`, `uiV6.perJourney`, `uiV6.polish`) so any wave could be killed independently.

Outcome: 42 active + 3 optional requirements scoped, 39 active requirements shipped across Phases 48-53, 3 optional (QUAD-*) deferred per Phase 53 SOFT-gate telemetry showing no quadrant-redesign signal.

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

- **Feature-flag gating mandatory** for every wave (`uiV6.landing`, `uiV6.leanTrim`, `uiV6.perJourney`, `uiV6.polish`). Each acts as an independent kill-switch — no big-bang shipping.
- **Wave ordering corrected** during planning — Wave 1 (Landing) strictly precedes Wave 2 (Lean trim); they cannot run in parallel (the landing redirects depend on routes still existing during rollout).
- **DB schema corrected pre-execution** — custom-dashboard audit uses `dashboard_layouts(layout jsonb)`, not the fictional `layout_versions.placements` from the v1 plan. Surfaced by Phase 48 pre-flight, fixed before Phase 51 LEAN-05 implementation.
- **`next.config.redirects[]`** is the chosen mechanism for deleted-route redirects (308 permanent, preserves query strings for deep-links like `/pm/wishes?tab=rejected`).
- **i18n keys land under `sidebar.personaSections.*`** to avoid collision with existing `sidebar.staff` / `sidebar.projects` section headings.
- **Widget-registry defensive fallback** — unknown widget IDs render a "Widget ej tillgänglig" placeholder, never throw. Protects tenant custom layouts during widget-deletion rollout.
- **LEGACY_LAYOUTS preserved** for flag-off rollback — DEFAULT_LAYOUTS holds the polished layout, LEGACY_LAYOUTS holds the pre-trim layout. Both ship; flag selects which renders. Verifier agent initial run was misled by grep hits on LEGACY_LAYOUTS — second run with explicit context passed.
- **SOFT gate over HARD gate for POLISH-07** — viewport-fit diagnostic emits `*.json` artifacts to CI rather than `expect()` calls, so a Phase 54 redesign can be triggered by data rather than a CI red.
- **Counter-proposal flow stays deferred** (inherited from v5.0 non-goals) — not in any v6.0 phase.

**Issues Resolved:**

- LM department-picker raw i18n keys rendering as primary text (UNBREAK-01/02 + new DepartmentPicker)
- PM Home stuck on loading spinner when `projects: []` (UNBREAK-03 guard reorder)
- /admin and /admin/people 500 errors from missing dev Neon migrations (UNBREAK-04/05)
- PersonaGate hardcoded "linjechefs-personan" copy (UNBREAK-06)
- 12 Playwright specs broken by route changes (UNBREAK-07)
- 7 dead widgets and 3 duplicate dashboard surfaces removed (LEAN-05/06/07 + LEAN-11 migration)
- Persona-blind notification bell + top-nav (POLISH-01/02)
- Manager + project-leader dashboard overflow on 1440×900 (POLISH-07 within threshold)

**Issues Deferred:**

- **Phase 54 (QUAD-*)** — deferred indefinitely; SOFT-gate telemetry shows no need
- **53-HUMAN-UAT Test 3** — operator migration rowcount check on prod DB (idempotence already verified on pglite; runbook captured in `.planning/runbooks/v6.0-uat-test-3-rowcount-check.md`)
- **52-HUMAN-UAT (6 pending)** — partial-pass status carried forward
- **Counter-proposal flow** — still deferred from v5.0
- **Mobile-first responsive pass** — out of scope for v6.0
- **Real role-based permissions** — personas remain UX shortcuts (ADR-004)
- **Email/Slack notification channel** — in-app only

**Technical Debt Incurred:**

- LEGACY_LAYOUTS code path still ships alongside DEFAULT_LAYOUTS — must be removed once all tenants are migrated off the flag-off path (cleanup work, post-stable-rollout).
- Three dev-env harness gaps logged but not fixed: `/api/test/seed` no-transactions under neon-http; `/api/v5/*/count` 404 in Turbopack dev; `requireRole()` bypasses Clerk session-level guards. None affect production.
- 4 review-fix batches required after the verified-clean ship (`53-fix` commits MJ-01 through UI-MN-04) — pattern of finishing in a worktree branch and needing fast-forward merges. Documented in v6.0 retrospective.
- Three one-shot SQL migrations (`20260422_polish_*`) are operator-applied, not part of the Drizzle migration runner — manual prod application required, runbook in `.planning/runbooks/`.

**Cross-cutting v6.0 patterns established:**

- "SOFT gate" pattern — emit diagnostic artifacts in CI for human review rather than failing builds, when the right action depends on user-facing telemetry rather than code correctness.
- "Flag-off parity invariant" pattern — every wave ships an automated test asserting the flag-off code path renders identically to pre-wave baseline. Allows rollback at any time.
- "LEGACY_* + DEFAULT_*" dual-layout pattern for safe layout swaps under feature flags.
