# Nordic Capacity -- v3.0 Roadmap: Switch from Excel

**Defined:** 2026-03-30
**Goal:** Make the app self-explanatory enough that a line manager can import their spreadsheet, see immediate value, and never go back to Excel.
**Approach:** No new features. UX clarity, role-based experience, and friction removal.

## Phases

- [ ] **Phase 18: Role-Based Landing Experience** - Route users to their role-appropriate view after login; heat map as the front door
- [ ] **Phase 19: Self-Explanatory Navigation** - Rename nav items to task-oriented labels; make each view's purpose obvious
- [ ] **Phase 20: Heat Map as Hero** - Surface overloaded people immediately; summary banner, click-to-edit, sort by severity
- [ ] **Phase 21: Import-to-Value Flow** - After import, redirect to heat map with success context; import accessible from heat map
- [ ] **Phase 22: Swedish Localization** - i18n framework, Swedish as primary language for all UI labels (stretch)

## Phase Details

### Phase 18: Role-Based Landing Experience
**Goal:** Users land on the view that answers their primary question, not a generic person list
**Depends on:** Phase 17 (v2.0 complete)
**Requirements:** UX-01, UX-02, UX-03
**Success Criteria:**
  1. After login, planners and viewers land on `/dashboard/team` (team load heat map)
  2. Admins/owners land on `/dashboard` (KPI overview)
  3. Heat map shows a prominent "Plan hours" action that links to the person input grid
  4. Root path `/` redirects based on role, not hardcoded to `/input`
**Key files:**
  - `src/app/(app)/page.tsx` or root redirect logic
  - `src/components/layout/app-shell.tsx`
  - `src/app/(app)/dashboard/team/page.tsx`

### Phase 19: Self-Explanatory Navigation
**Goal:** A first-time user can find the right tab without explanation
**Depends on:** Phase 18
**Requirements:** UX-04, UX-05, UX-06
**Success Criteria:**
  1. Nav labels use task-oriented names (not generic "Input", "Data", "Dashboard")
  2. Proposed renames: Input -> "Planera" / "Plan Hours", Dashboard -> "Oversikt" / "Overview", Dashboard>Team -> "Teambelastning" / "Team Load", Alerts -> "Varningar" / "Warnings", Data -> "Exportera" / "Export"
  3. Each nav item has a brief description or subtitle visible on hover/first visit
  4. A line manager tested by the operator can find "see my team's load" and "edit hours" without guidance
**Key files:**
  - `src/components/layout/top-nav.tsx`
  - `src/components/layout/side-nav.tsx`

### Phase 20: Heat Map as Hero
**Goal:** The heat map answers "who's overloaded?" in the first 2 seconds
**Depends on:** Phase 18
**Requirements:** UX-07, UX-08, UX-09, UX-10
**Success Criteria:**
  1. Overloaded people are sorted to the top by default
  2. A summary banner shows count of overloaded people and the most critical month (e.g., "3 personer overbokade i april")
  3. Person names in the heat map are clickable -> opens their planning grid
  4. Color coding is unmistakable at a glance (current thresholds validated or adjusted)
**Key files:**
  - `src/app/(app)/dashboard/team/page.tsx`
  - Heat map table component (under `src/features/` or `src/components/`)

### Phase 21: Import-to-Value Flow
**Goal:** After importing data, the user immediately sees the payoff
**Depends on:** Phase 18, Phase 20
**Requirements:** UX-11, UX-12, UX-13
**Success Criteria:**
  1. After successful import, user is redirected to team heat map with a success banner showing count of imported people
  2. Heat map page has an "Import from Excel" button (not only accessible via Data/Export page)
  3. The transition from "I uploaded my spreadsheet" to "I can see my team's load" takes < 3 clicks
**Key files:**
  - Import wizard final step / success handler
  - `src/app/(app)/dashboard/team/page.tsx`

### Phase 22: Swedish Localization (Stretch)
**Goal:** Remove language friction for Swedish-speaking end users
**Depends on:** Phase 19 (nav labels are the highest-impact strings)
**Requirements:** UX-14, UX-15
**Success Criteria:**
  1. i18n framework (next-intl or similar) is configured
  2. All user-facing labels, nav items, alert messages, and button text have Swedish translations
  3. Swedish is the default locale; English is fallback
  4. Platform admin pages remain in English (operator tools)
**Key files:**
  - New: `src/i18n/` or `src/messages/` directory
  - All component files with user-facing strings

## What v3.0 is NOT

- No new features, data models, or API endpoints
- No enterprise features (SSO, billing, API, integrations)
- No mobile optimization (desktop is the primary use case)
- No new dashboard widgets or chart types
- No backend performance work (unless blocking UX goals)

## Success Test

A line manager at one of the operator's client organizations can:
1. Import their existing Excel planning spreadsheet
2. Immediately see a heat map showing who's overloaded
3. Click a person to edit their hours in a familiar grid
4. Come back the next day and land on the heat map showing current state
5. Do all of this without someone explaining what each tab does
